API Guide

  • Choose a technology that is most suitable for your API needs
  • Create your API solution and push it to a GitHub repository

    • Naming your API should be data specific

  • Familiarize yourself with the Hackney Development Standards
  • Set up your project structure, including Swagger set up.
  • Following the API Playbook , start developing new endpoints while also having the Hackney Development Standards in mind
  • While building a new endpoint, ensure you are submitting smaller, but frequent GitHub pull requests. It is essential that pull requests are short to ensure that other developers do not need to take a big part of their time reviewing your pull request.
  • Always run your tests prior to submitting a Pull Request.
  • Before merging any code, ensure pull request is approved by at least two other developers.
  • As per the API playbook, ensure your API has Continous Integration set up as well as automated testing.
  • Every new API endpoint needs to be documented following the White House standards in a GitHub Read me file.
